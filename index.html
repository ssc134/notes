Time complexity glossary
Big-O/Ω/Θ, worst/best/average-case, and amortized time
📐 The mathematical notations (the bounds)
  • Big O (O): the upper bound. The algorithm’s time will grow no faster than this rate.
  • Big Ω (Ω): the lower bound. The algorithm’s time will take at least this long to grow.
  • Big Θ (Θ): the tight bound. The exact growth rate (runtime is sandwiched by this function).
  • Little o (o) & little ω (ω): strict upper and lower bounds (runtime must be strictly less than or greater than the bound, never equal).
⏱️ The scenarios (the inputs)
  • Worst-Case: the single specific input of size n that takes the absolute maximum time. The industry standard is to bound this using O.
  • Best-Case: the single specific input of size n that takes the absolute minimum time. Mathematically, this is best described using Ω.
  • Average-Case: the expected time across all possible inputs of size n, weighted by the probability of each input occurring.
🏦 Amortized complexity (the sequence)
  • Amortized Time: the guaranteed average cost per operation over a long sequence. Occasional expensive operations (like an array copying itself to resize) are “paid for” by a large number of cheap operations. It ignores probability.
